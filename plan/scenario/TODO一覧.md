## Why

| As：ペルソナ | I want：～したい   | So that：そうすることで、～からだ          |
| ------- | ------------- | ----------------------------- |
| ユーザーは   | 自身のTODOを一覧したい | そうすることで、TODOへの対応や管理がしやすくなるからだ |
## Acceptance Criteria

| Given：～の場合 | When：～の時        | Then：～なる                     |
| ---------- | --------------- | ---------------------------- |
| -          | 登録されているTODOがない時 | 登録されているTODOがないことを伝える画面が表示される |
| -          | 登録されているTODOがある時 | 登録されているTODOの一覧が表示される         |
| 未ログイン状態の場合 | 一覧を表示しようとした時    | ログインフォームにリダイレクトする            |
| -          | 削除ボタンをクリックした時   | そのTODOが削除される                 |
## 不明事項
- 文字数が多いタイトルの表示方法
- ステータスや締め切り変更後のリスト表示（絞り込みに引っかからなくなった場合、一覧から見えなくするか？）
- 一覧状態でのTODO削除時の挙動
## 代替案
- タイトルに入力できる文字数を制限する
## 判断メモ
- 画面に表示するもの
	- 絞り込みフォーム（ページネーション含む）
	- 新規作成ボタン
	- 各TODO
		- 情報
			- タイトル
			- 作成日時
		- 操作要素
			- 削除ボタン
			- ステータス変更
			- 詳細ボタン
			- 締め切り
